4.5 Updating Specific Software Packages

After a development environment is created, instead of updating the entire environment, packages may require individual updates or other modifications. This includes advanced scenarios, such as troubleshooting or updating installed software to specific versions. In this case, Nintendo Dev Interface (NDI) is able to separately manage the installation of packages in each environment. Unlike when updating complete environments, a package may be individually updated to any version.

To manage the software in an environment, follow these steps.

  1. Start NDI and select Dev Environments.
  2. The Dev Environments screen is displayed. In the list on the left, select the development environment to modify.
  3. Click the Manage button and select Manage Details from the menu.



  4. The Configure Environment screen displays all software currently installed. Select an option from the version drop-down list box for the desired package to change its installation status. In most cases, this will involve the selection of a different software version, but may also include additional types of modification, such as uninstalling or reinstalling. As follows, a symbol will be associated with most selections for reference.
    1. The upgrade icon () indicates that a later version of the software is available. 
    2. The install icon () is displayed when the selected version will be downloaded and installed.
    3. The required icon () is displayed if a software package is required to be selected or installed in order for another selected package to install.
    4. The reinstall icon () is displayed when a software package is selected to be reinstalled. This option is not available for all software.
    5. The uninstall icon () is displayed when a software package is selected to be removed from the system. This option is not available for all software. Uninstallation of software may prevent other software from functioning correctly. In this case, a warning will be displayed as installation proceeds.
    6. The rollback icon () is displayed when the rollback option is selected. This option is available if a software package has been previously updated. It reverts the software package from the current version to the previously installed version.
    7. Selections in red are incompatible with other selections.
    8. For more information, see the Manage Details subsection in the User Interface Reference.



  5. After making all version selections, click Next. The installation summary screen is displayed, indicating the changes that will be made to the development environment.
  6. Click Apply Changes to begin the update process. After the update is complete, NDI returns to the Dev Environments screen.
