4.10 Exporting a Development Environment (MegaZarf Format)

Nintendo Dev Interface (NDI) cannot directly copy an environment that you have created. However, an environment can be stored in a file named a "MegaZarf", which can then be distributed to other users or used on your system to create a new instance of the original environment. A MegaZarf contains information about all packages that are currently installed, but will not record any changes that have been made outside of NDI, such as modifications through installed software or external programs.

This can be helpful when you need to distribute a copy of an environment to your team, but do not want to have the team take additional time downloading files, or to share an environment by putting the MegaZarf in a common location.

To create a MegaZarf, follow these steps.

  1. Start NDI and select Dev Environments.
  2. Select the environment to be exported from the list on the left.
  3. Click the Manage button and select Export from the menu.



  4. On the Export Environment screen, enter the name for the MegaZarf in the Export File Name field. Click the Browse button to select the location the MegaZarf will be exported to. 
  5. In the Description field, provide an explanation about the use of the MegaZarf. This will be visible when other users import the MegaZarf.



  6. Click Export. NDI processes the environment and creates the new MegaZarf at the location that was indicated in step 4.
